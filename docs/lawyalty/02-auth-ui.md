```

│         라윌티 계정으로 시작하세요          │  ← 14px muted, 24px below
│                                          │
│   이메일                                   │  ← label 13px / 500
│   ┌────────────────────────────────────┐ │
│   │ name@example.com                    │ │  ← input 44px h, 15px font
│   └────────────────────────────────────┘ │
│                                          │
│   비밀번호                            [표시] │
│   ┌────────────────────────────────────┐ │
│   │ ••••••••                            │ │
│   └────────────────────────────────────┘ │
│                                          │
│   ☐  이 기기 기억하기      비밀번호 찾기 →  │  ← 13px
│                                          │
│   ┌────────────────────────────────────┐ │
│   │           로그인                     │ │  ← Primary button 48px h
│   └────────────────────────────────────┘ │
│                                          │
│   ─── 또는 ───                            │  ← 12px muted divider
│                                          │
│   [Google 로 로그인]                       │  ← outline glass button
│   [Kakao 로 로그인]                        │
│                                          │
│   계정이 없으신가요? 가입하기 →             │  ← 13px, 잉크 색
│                                          │
└──────────────────────────────────────────┘
```

### 필드 스펙

| 필드 | type | 필수 | validation | placeholder |
|---|---|---|---|---|
| email | email | ✅ | RFC 5322, 254자 이하 | `name@example.com` |
| password | password | ✅ | 1자 이상 (서버에서 추가 검증) | (없음) |
| remember | checkbox | ❌ | — | — |

### 동작
- 제출 → `POST /auth/login {email, password, remember}`
- 응답:
  - `200` → 세션 쿠키 set, `/` 또는 `?redirect=` 로 이동
  - `401 invalid_credentials` → 인라인 에러 (전체 폼 위), focus email
  - `403 unverified_email` → `/auth/verify-email` 리다이렉트
  - `403 pending_approval` → `/auth/pending` 리다이렉트
  - `423 locked` → "보안을 위해 잠시 후 다시 시도하세요. (X분 후 자동 해제)"
  - `200 mfa_required` → `/login/mfa?challenge=xxx` 리다이렉트

### 인터랙션
- email 인풋 blur 시 형식 검증 (인라인 마이크로카피)
- 비밀번호 [표시] 토글
- Caps Lock 켜져 있으면 input 아래 노란색 작은 안내
- Submit 중 버튼 disabled + spinner
- 에러 발생 시 `aria-live="polite"` 영역에 메시지

### 필드 스펙

| 필드 | 필수 | validation | 비고 |
|---|---|---|---|
| name | ✅ | 2~30자, 한글·영문·공백 | |
| email | ✅ | RFC 5322, 254자, 중복 체크 (debounced 500ms) | |
| password | ✅ | 10자 이상 + 영/숫/특 중 2종 + HIBP top-1000 차단 | strength bar |
| role | ✅ | enum: lawyer / broker / architect / user | drop-down |
| license_number | ❌ | 자격별 형식 (서버 검증) | role=user 면 숨김 |
| organization | ❌ | 1~80자 | |
| terms_agreed | ✅ | true | |
| marketing_agreed | ❌ | bool | |

### 동작
- 제출 → `POST /auth/signup`
- 성공 → `/auth/verify-email` (인증 메일 안내)
- 실패:
  - `409 email_in_use` → email 인풋 아래 인라인
  - `400 weak_password` → password 인풋 아래
  - `429 rate_limited` → 폼 위 알림 (X분 후 다시 시도)

### 마이크로 인터랙션
- email 인풋 typing → 500ms debounce 후 `HEAD /auth/check-email?email=` (가용성 표시 ✓ 또는 ✗)
- 직무 = "변호사·법무사" 선택 시 자격번호 placeholder 가 "변호사 등록번호 (예: 12345)" 로 변경
- terms 체크 필수 — uncheck 면 submit 비활성

## 9. 화면 7: MFA 설정 (`/settings/security/mfa`)

```
2단계 인증 (MFA) 설정

1. Authenticator 앱을 여세요 (1Password, Google Authenticator 등)
2. 아래 QR 을 스캔하거나 키를 직접 입력하세요

   [QR 코드 200×200]
   또는: ABCD EFGH IJKL MNOP

3. 앱에 표시된 6자리 코드를 입력하세요

   ┌─┬─┬─┬─┬─┬─┐
   │ │ │ │ │ │ │  ← OTP 6자리, 자동 advance
   └─┴─┴─┴─┴─┴─┘

   [활성화]

4. 백업 코드 8개 (한 번만 표시 — 안전하게 보관하세요)
   X4F2-K9LM    P3D7-N2QW
   ...
   [복사] [다운로드]
```

- TOTP secret 발급 → QR (otpauth:// URI)
- code 입력 검증 후 `mfa_enabled=1`
- 백업 코드 8개 발급 — 한 번만 표시, hash 만 저장

## 12. 카피·에러 메시지

### 카피 톤
- **공식적이지만 따뜻함** — "~합니다", "~해 주세요" 일관
- 영어·약어 노출 금지: `2FA` → `2단계 인증`, `OTP` → `인증 코드`, `Resend` → `다시 보내기`
- 행동 동사로 시작하는 버튼: "로그인", "가입 신청", "재설정 링크 받기", "이메일 인증"

### 주요 에러 메시지

| 코드 | 화면 메시지 |
|---|---|
| `invalid_credentials` | 이메일 또는 비밀번호가 올바르지 않습니다. |
| `unverified_email` | 이메일 인증이 필요합니다. 메일을 확인해 주세요. |
| `pending_approval` | 관리자 승인 대기 중입니다. 평균 1~2 영업일 소요됩니다. |
| `account_locked` | 보안을 위해 잠시 잠겼습니다. 약 X분 후 다시 시도해 주세요. |
| `email_in_use` | 이미 가입된 이메일입니다. [로그인](/login) |
| `weak_password` | 비밀번호가 너무 약합니다. 10자 이상, 영문·숫자·특수문자 중 2종류 이상 사용해 주세요. |
| `rate_limited` | 너무 많은 요청이 있었습니다. X분 후 다시 시도해 주세요. |
| `mfa_invalid_code` | 인증 코드가 올바르지 않습니다. 다시 시도해 주세요. |
| `token_expired` | 링크가 만료되었습니다. 다시 요청해 주세요. |
| `network_error` | 네트워크 연결을 확인해 주세요. |
| `unknown_error` | 일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요. |

## 13. 접근성 체크리스트

- [ ] 모든 폼 필드에 `<label>` 명시 (시각적 숨김 X)
- [ ] 에러는 `aria-live="polite"` 영역에 publish
- [ ] 인풋 invalid 시 `aria-invalid="true"` + `aria-describedby` → 에러 메시지 id
- [ ] 색상만으로 정보 전달 X (에러 메시지 + 아이콘 같이)
- [ ] 키보드만으로 모든 화면 통과 가능
- [ ] focus ring 잉크 2px outline + 2px offset
- [ ] 비밀번호 [표시] 토글에 `aria-pressed`
- [ ] OTP 인풋 6칸은 `inputmode="numeric"` `autocomplete="one-time-code"`
- [ ] 폰트 최소 13px, 본문 15~17px
- [ ] 로고는 `alt="라윌티"`, 데코레이션 SVG 는 `aria-hidden`
